### ЭТАП 5: Риски и правовые аспекты

#### Блок 5.1: Управление рисками

**Промпт**:
```
БЛОК 5.1 — ПРОВЕРКА УПРАВЛЕНИЯ РИСКАМИ

Оцени идентификацию и mitigation рисков проекта.

ШАГ 1: Проверь наличие анализа рисков:

| Категория рисков | Наличие в ТЗ | Раздел ТЗ | Идентифицированные риски | Меры mitigation | Достаточность |
|-----------------|--------------|-----------|------------------------|----------------|--------------|
| Технические (миграция данных) | | | | | |
| Технические (интеграция) | | | | | |
| Организационные (внедрение) | | | | | |
| Временные (сроки этапов) | | | | | |
| Кадровые (компетенции команды) | | | | | |
| Финансовые (бюджет) | | | | | |
| Юридические (лицензии, НПА) | | | | | |

ШАГ 2: Для критичных рисков (миграция, интеграция с legacy) проверь:
- Определён ли план B (fallback)?
- Есть ли пилотное внедрение перед масштабированием?
- Предусмотрено ли поэтапное замещение (а не "big bang")?

ШАГ 3: Проверь реалистичность графика:
- Соответствуют ли сроки сложности задач?
- Учтены ли риски задержек?
- Предусмотрены ли буферы времени?

ФОРМАТ ОТВЕТА: Таблица рисков + оценка реалистичности планирования.
```
