**Промпт**:
```
БЛОК 3.1 — ПРОВЕРКА ОТРАСЛЕВОЙ СПЕЦИФИКИ

Проверь учёт стандартов и требований отрасли (используй загруженный контекст).

ШАГ 1: Для автомобилестроения проверь ссылки на отраслевые стандарты:

| Стандарт | Область применения | Упоминание в ТЗ (Да/Нет) | Раздел ТЗ | Реализация в требованиях | Критичность |
|----------|-------------------|--------------------------|-----------|-------------------------|------------|
| ISO 26262 | Функциональная безопасность автомобилей | | | | ВЫСОКАЯ |
| IATF 16949 | Система менеджмента качества в автопроме | | | | ВЫСОКАЯ |
| STEP AP242 | Обмен данными CAD/PLM | | | | СРЕДНЯЯ |
| VDA (серия) | Стандарты немецких автопроизводителей | | | | СРЕДНЯЯ |
| AUTOSAR | Архитектура ПО автомобилей | | | | зависит от контекста |

ШАГ 2: Проверь специфические функции для автопрома:
- Многоуровневые BOM (EBOM, MBOM, SBOM)
- Управление вариативностью (опции комплектации)
- Интеграция с ERP для производства
- FMEA-анализ
- Trace-матрицы требований

ШАГ 3: Проверь требования к интеграции с CAD:
- Поддерживаемые форматы (JT, STEP AP242, Parasolid, ACIS)
- Ассоциативность связей CAD ↔ PLM

ФОРМАТ ОТВЕТА: Таблица стандартов + список недостающих отраслевых функций.
```
