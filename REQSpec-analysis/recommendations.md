## Режим генерации исправлений (отдельно!)

**Когда использовать**: ТОЛЬКО после завершения аудита и утверждения реестра замечаний.

**Промпт для генерации**:
```
РЕЖИМ ГЕНЕРАЦИИ ИСПРАВЛЕНИЙ

Для замечаний из утверждённого реестра с приоритетом [ВЫСОКИЙ/СРЕДНИЙ] сгенерируй конкретные формулировки для внесения в ТЗ.

ВХОДНЫЕ ДАННЫЕ:
- ID замечания: [указать]
- Текущая формулировка: [цитата из ТЗ]
- Описание проблемы: [из реестра]

ТРЕБОВАНИЯ К ГЕНЕРАЦИИ:
1. Формулировка должна соответствовать стилю ТЗ
2. Обязательная ссылка на конкретный пункт ГОСТ (если применимо)
3. Измеримые критерии (числа, сроки, проценты)
4. Проверяемость при приёмке

ФОРМАТ ОТВЕТА:
- Предлагаемая формулировка для ТЗ
- Обоснование (почему именно так)
- Как проверить выполнение при приёмке

ВАЖНО: Это ПРЕДЛОЖЕНИЕ для обсуждения, а не окончательный текст!
```
